# Wedding site — Аскар & Анель

Готовый одностраничный wedding invitation на чистом HTML/CSS/JS с более «editorial / Tilda-like» подачей: крупная обложка, воздушные отступы, мягкие reveal-анимации при скролле, countdown, календарь, карта, галерея и RSVP-форма.

## Что внутри

- `index.html` — разметка сайта
- `styles.css` — mobile-first стили
- `script.js` — reveal-анимации, countdown, клиентская валидация и AJAX-отправка формы
- `worker.js` — Cloudflare Worker для безопасной отправки в Telegram Bot API
- `assets/*.webp` — лёгкие декоративные WebP-заглушки под hero и галерею

## Основная стилистика версии

- полноэкранная обложка с мягким overlay
- рукописные заголовки с кириллическим fallback
- плавные появления блоков на скролле через `IntersectionObserver`
- лёгкий parallax только для hero-фона без тяжёлых библиотек
- countdown до даты свадьбы
- полупрозрачные glass-card карточки
- спокойная типографика и большие отступы в духе современных свадебных invite-лендингов

## Что заменить перед публикацией

В `index.html`:
- название ресторана и адрес
- ссылку в блоке карты (`href="https://go.2gis.com/"`) на реальную route-ссылку 2GIS
- при необходимости текст приглашения
- город / время, если нужны другие значения

В `assets/`:
- замените `hero.webp`, `gallery-1.webp`, `gallery-2.webp` на ваши фотографии
- оптимальные форматы: WebP, разумное сжатие, ширина hero 1600–2000px, галерея 1000–1600px

В `script.js`:
- при необходимости поменяйте `FORM_ENDPOINT`, если Worker размещён не на `/api/rsvp`
- при необходимости скорректируйте `WEDDING_DATE`

## Подключение Cloudflare Worker

### 1) Создайте Worker
Разместите `worker.js` как отдельный Worker или через Pages Functions/Workers route.

### 2) Добавьте секреты и переменные
Добавьте:
- `BOT_TOKEN` — токен Telegram-бота (как secret)
- `CHAT_ID` — ваш chat_id
- `ALLOWED_ORIGIN` — домен сайта, например `https://invite.example.com`

### 3) Настройте роут
Сделайте так, чтобы фронтенд отправлял POST-запросы на `/api/rsvp`.

### 4) Проверка
После деплоя:
- откройте сайт
- отправьте форму
- проверьте, что сообщение пришло в Telegram

## Замечания по UX и производительности

- mobile-first, диапазон 320–1440 px
- нет горизонтального скролла
- все интерактивные элементы не ниже 44px
- анимации отключаются при `prefers-reduced-motion`
- hero использует `100svh`, что стабильнее для iOS Safari
- изображения уже в WebP
- без тяжёлых фреймворков и без iframe-карты
